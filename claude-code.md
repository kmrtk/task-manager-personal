# Claude Code 学習記録

## モードについて

### Plan mode（計画モード）
**何をするときに使う：**
- 既存のアプリに大きな変更を加える前に、方針を決めるとき
- 複数ファイルにまたがる修正の内容をClaudeとすり合わせしたいとき
- 「何をどう変えるか」を実行前に確認・合意しておきたいとき

**特徴：**
- Claudeはファイルを読むだけで、実際の編集は行わない
- 「こう変えようと思うがどうか？」と全体の計画を先に提示してくれる
- 承認してから初めて実行される

> **ひとことで言うと：方針を決めるモード**

---

### アスクモード（通常モード）
**何をするときに使う：**
- Plan modeで方針を決めた後、実際に実行するとき
- 都度確認しながら進めたいとき

**特徴：**
- ツール実行のたびに「やっていい？」と確認してくれる
- Claudeが自発的に判断した内容を一手ずつ承認して進める

> **ひとことで言うと：都度確認しながら実行するモード**

---

### エディタモード（自動承認モード）
**何をするときに使う：**
- 方針が明確で、Claudeに自動的に進めてほしいとき
- 細かい確認なしにテンポよく作業を進めたいとき

**特徴：**
- Claudeが自動で判断・実行する
- 承認なしで進むので、信頼できる作業内容のときに使う

> **ひとことで言うと：自動で実行するモード**

---

## 推奨の使い方（フロー）

```
① Plan mode で方針を確認・合意
        ↓
② アスクモード に切り替えて実行（都度承認）
```

この2段階で「意図しない変更」を未然に防げる。

---

## モード切り替え

`Shift + Tab` で順番に切り替え可能。
