## 3. クロコの意見・気になった点

### ① タグの連携が分断されている
設定画面でタグを追加しても、タスク詳細のタグ選択肢は別データで管理している。
本実装では設定で作ったタグがそのままタスクに使えるよう統一する必要がある。

### ② F07（グループ並び替え）は実装難易度が低いので追加してもよいかも
タスクのドラッグ&ドロップはSortable.jsで実装済みなので、グループにも同じ仕組みを適用するだけ。
プロトタイプに追加して動きを確認してみてもよいと思う。

### ③ F20（完了表示）はUXに関わるので早めに決めておきたい
「完了」はチェックボックスで操作するのか、タスクカード上のボタンなのか、
詳細画面で操作するのかが要件定義書に書かれていない。明日すり合わせたい。

### ④ コメント機能（F22〜F24）は本当に必要か再確認したい
個人利用のアプリなのでメモ欄（F09）で代替できる可能性がある。
コメントとメモの違いを明確にしないと実装したあとに「いらなかった」となりそう。

### ⑤ F31（作成日）の表示場所を決めたい
タスクカード上に表示するのか、詳細画面だけに表示するのかが未定。

## 5. すり合わせ結果（2026-06-11）

| ID | 機能 | 決定内容 |
|----|------|---------|
| F07 | グループ並び替え | スコープ外。カラムは「未着手/進行中/完了」の3種類固定 |
| F14 | タグ編集 | スコープ外。将来追加予定 |
| F20 | 完了表示 | タスクカードに「未着手/進行中/完了」ボタンを設置。ボタン押下で対応カラムに自動移動。ドラッグ&ドロップとの併用可 |
| F22〜F24 | コメント機能 | 削除。メモ（F09）で代替 |
| F25〜F27 | チェックリスト | 削除。メモ（F09）で代替 |
| F29 | タスク復元 | 実装済み |
| F31 | 作成日表示 | スコープ外（要件定義書から削除済み） |
| F32 | カラム内タスク並び替え | 実装済み。各カラムに「日付順」「優先度順」ボタンを追加。押すたびに昇順→降順がループする |
| F33 | タスク詳細の保存・削除ボタン | スコープ外（要件定義書から削除済み）。「キャンセル/追加・更新」ボタンのモーダル方式で対応 |
