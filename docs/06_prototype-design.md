# プロトタイプ設計書　タスク管理アプリ（Trello風）

---

## 1. 概要

HTML / CSS / JavaScript を使ってアプリの主要画面と動きを確認するためのモック。
画面遷移・タスク操作・ドラッグ＆ドロップの動きを体感することが目的。
データの保存は行わない（ページを再読み込みすると初期状態に戻る）。

---

## 2. ファイル構成

```
src/
├── index.html        # フォルダ一覧画面
├── folder.html       # フォルダ画面（グループ・タスク）
├── task.html         # タスク詳細画面
├── settings.html     # 設定画面
├── css/
│   └── style.css     # 全画面共通スタイル
└── js/
    ├── app.js        # 画面ごとの操作処理
    └── drag.js       # ドラッグ＆ドロップ処理
```

---

## 3. 使用技術

| 種類 | 技術 |
|------|------|
| マークアップ | HTML5 |
| スタイル | CSS3 |
| 動作 | JavaScript（バニラJS） |
| ドラッグ＆ドロップ | Sortable.js（CDN） |
| データ | なし（ページ内の仮データのみ） |

## 4. 画面ごとの設計

### 4-1. フォルダ一覧画面（index.html）

**レイアウト：**
```
┌─────────────────────────────────┐
│  ヘッダー（アプリ名 ＋ 設定ボタン）   │
├─────────────────────────────────┤
│  ┌──────┐ ┌──────┐ ┌────────┐  │
│  │仕事   │ │趣味   │ │＋ 追加 │  │
│  └──────┘ └──────┘ └────────┘  │
└─────────────────────────────────┘
```

**実装する動き：**
- フォルダカードをクリック → folder.html に遷移
- 「＋ 追加」ボタン → 入力フィールドが表示され、フォルダ名を入力して追加
- フォルダ名をダブルクリック → 編集モードに切り替え（インライン編集）

---

### 4-2. フォルダ画面（BoardPage）

**レイアウト：**
```
┌──────────────────────────────────────────────┐
│  ← 戻る  │  フォルダ名  │  [検索バー]  │ ゴミ箱 │
├──────────────────────────────────────────────┤
│ ┌──────────┐ ┌──────────┐ ┌──────────┐      │
│ │未着手 (3) │ │進行中 (1) │ │完了   (0) │      │
│ │[↑日付][↑優先度]│ │[↑日付][↑優先度]│ │[↑日付][↑優先度]│  │
│ │          │ │          │ │          │      │
│ │[タスクA]  │ │[タスクC]  │ │          │      │
│ │[タスクB]  │ │          │ │          │      │
│ │…+N件     │ │          │ │          │      │
│ │          │ │          │ │          │      │
│ │  ＋ 追加 │ │  ＋ 追加 │ │  ＋ 追加 │      │
│ └──────────┘ └──────────┘ └──────────┘      │
└──────────────────────────────────────────────┘
```

**実装する動き：**
- タスクカードをドラッグ → カラム間・カラム内で移動（dnd-kit）
- 「＋ 追加」ボタン → タスク作成モーダルを表示
- タスクカードをクリック（編集ボタン） → タスク編集モーダルを表示
- カラムは「未着手 / 進行中 / 完了」の3種類固定（ユーザーによる追加・削除不可）
- 検索バーでタスク名をリアルタイム検索
- 各カラムは最大5件表示。5件超えの場合はバッジ表示 → クリックで展開

---

### 4-3. タスク詳細モーダル（TaskModal）

> ※ 当初は別画面（task.html）として設計していたが、実装時にモーダルダイアログに変更。

**レイアウト：**
```
┌─────────────────────────────────┐
│  タスクを追加 / タスクを編集      │
├─────────────────────────────────┤
│  タスク名：[テキスト入力]          │
│                                 │
│  メモ：[テキストエリア]            │
│                                 │
│  ステータス：[プルダウン]          │
│                                 │
│  優先度：[HIGH][MEDIUM][LOW][なし] │
│                                 │
│  期限：[日付入力]                 │
│                                 │
│  タグ：[入力] + 追加 / [タグ ×]   │
│                                 │
│          [キャンセル] [追加/更新]  │
└─────────────────────────────────┘
```

**実装する動き：**
- タスク名・メモ・ステータス・優先度・期限・タグを入力・編集できる
- 「追加 / 更新」ボタンでAPIを呼び出して保存し、モーダルを閉じる
- 「キャンセル」ボタンでモーダルを閉じる（変更は破棄）
- タスクカードのステータスボタンをクリックで TODO → IN_PROGRESS → DONE とサイクル変更

---

### 4-4. 設定画面（settings.html）

**レイアウト：**
```
┌────────────────────────────┐
│  ← 戻る  │  設定            │
├────────────────────────────┤
│  ■ タグ管理                 │
│  [仕事 ×] [急ぎ ×] ＋追加   │
│                            │
│  ■ ゴミ箱                   │
│  [タスクA] [復元] [完全削除] │
│  [タスクB] [復元] [完全削除] │
└────────────────────────────┘
```

**実装する動き：**
- タグの追加・削除
- ゴミ箱のタスク一覧表示
- 「復元」ボタン → 元のグループに戻す
- 「完全削除」ボタン → 確認後に削除

---

## 5. 実装しない機能（プロトタイプ対象外）

| 機能 | 理由 |
|------|------|
| データの保存 | 動きの確認が目的のため不要 |
| サーバー・DB連携 | 本実装フェーズで対応 |
| ユーザー認証 | 個人利用前提のため不要 |
| アラートの自動更新 | 本実装フェーズで対応 |
