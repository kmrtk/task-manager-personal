# プロトタイプ設計書　タスク管理アプリ（Trello風）

---

## 1. 概要

HTML / CSS / JavaScript を使ってアプリの主要画面と動きを確認するためのモック。
完全な実装ではなく、画面遷移・タスク操作・ドラッグ＆ドロップの動きを体感することが目的。

---

## 2. ファイル構成

```
src/
├── index.html        # フォルダ一覧画面
├── folder.html       # フォルダ画面（グループ・タスク）
├── task.html         # タスク詳細画面
├── settings.html     # 設定画面
├── css/
│   └── style.css     # 全画面共通スタイル
└── js/
    ├── app.js        # 画面共通の処理（データ管理など）
    └── drag.js       # ドラッグ＆ドロップ処理
```

---

## 3. 使用技術

| 種類 | 技術 |
|------|------|
| マークアップ | HTML5 |
| スタイル | CSS3 |
| 動作 | JavaScript（バニラJS） |
| ドラッグ＆ドロップ | Sortable.js（CDN） |
| データ保存 | localStorage（モック用の仮データ） |

## 4. 画面ごとの設計

### 4-1. フォルダ一覧画面（index.html）

**レイアウト：**
```
┌─────────────────────────────────┐
│  ヘッダー（アプリ名 ＋ 設定ボタン）   │
├─────────────────────────────────┤
│  ┌──────┐ ┌──────┐ ┌────────┐  │
│  │仕事   │ │趣味   │ │＋ 追加 │  │
│  └──────┘ └──────┘ └────────┘  │
└─────────────────────────────────┘
```

**実装する動き：**
- フォルダカードをクリック → folder.html に遷移
- 「＋ 追加」ボタン → 入力フィールドが表示され、フォルダ名を入力して追加
- フォルダ名をダブルクリック → 編集モードに切り替え（インライン編集）

---

### 4-2. フォルダ画面（folder.html）

**レイアウト：**
```
┌──────────────────────────────────────────┐
│  ← 戻る  │  フォルダ名  │  設定ボタン       │
├──────────────────────────────────────────┤
│ ┌────────┐ ┌────────┐ ┌──────────────┐  │
│ │未着手   │ │進行中   │ │ ＋ グループ追加│  │
│ │        │ │        │ └──────────────┘  │
│ │[タスクA]│ │[タスクC]│                   │
│ │[タスクB]│ │        │                   │
│ │        │ │        │                   │
│ │＋ 追加  │ │＋ 追加  │                   │
│ └────────┘ └────────┘                   │
└──────────────────────────────────────────┘
```

**実装する動き：**
- タスクカードをドラッグ → グループ間・グループ内で移動（Sortable.js）
- 「＋ 追加」ボタン → タスク名の入力フィールド表示 → タスクを追加
- タスクカードをクリック → task.html に遷移
- 「＋ グループ追加」ボタン → グループ名を入力して追加

---

### 4-3. タスク詳細画面（task.html）

**レイアウト：**
```
┌─────────────────────────────┐
│  ← 戻る                      │
├─────────────────────────────┤
│  タスク名（編集可）            │
│                             │
│  期限：[カレンダー入力]         │
│  優先度：[色選択]              │
│  タグ：[タグ選択]              │
│                             │
│  メモ：[テキストエリア]         │
│                             │
│  チェックリスト               │
│  □ サブタスク1               │
│  □ サブタスク2               │
│  ＋ 項目を追加                │
│                             │
│  [ゴミ箱に移動]               │
└─────────────────────────────┘
```

**実装する動き：**
- タスク名・メモを直接編集できる（入力後に自動保存）
- 期限日をカレンダーUIで設定
- チェックリストの項目を追加・チェック操作
- 「ゴミ箱に移動」ボタン → フォルダ画面に戻る

---

### 4-4. 設定画面（settings.html）

**レイアウト：**
```
┌────────────────────────────┐
│  ← 戻る  │  設定            │
├────────────────────────────┤
│  ■ タグ管理                 │
│  [仕事 ×] [急ぎ ×] ＋追加   │
│                            │
│  ■ ゴミ箱                   │
│  [タスクA] [復元] [完全削除] │
│  [タスクB] [復元] [完全削除] │
└────────────────────────────┘
```

**実装する動き：**
- タグの追加・削除
- ゴミ箱のタスク一覧表示
- 「復元」ボタン → 元のグループに戻す
- 「完全削除」ボタン → 確認後に削除

## 5. データ管理（localStorage）

モック用の仮データをブラウザの localStorage に保存する。
サーバーやDBは使わず、ブラウザを閉じても状態が維持される。

**データ構造（イメージ）：**
```json
{
  "folders": [
    {
      "id": 1,
      "name": "仕事",
      "groups": [
        {
          "id": 1,
          "name": "未着手",
          "tasks": [
            {
              "id": 1,
              "title": "タスクA",
              "memo": "",
              "deadline": "2026-06-30",
              "priority": "red",
              "tags": [1],
              "checklist": [],
              "deleted": false,
              "createdAt": "2026-06-10"
            }
          ]
        }
      ]
    }
  ],
  "tags": [
    { "id": 1, "name": "急ぎ", "color": "red" }
  ]
}
```

## 6. 実装しない機能（モック対象外）

以下はモックでは省略し、本実装フェーズで対応する。

| 機能 | 理由 |
|------|------|
| サーバー・DB連携 | モックはlocalStorageで代替 |
| ユーザー認証 | 個人利用前提のため不要 |
| アラートの自動更新 | 画面表示時に一度だけ判定すれば十分 |
| タスク並び替え（細かい順序保存） | ドラッグの動きの確認が目的のため簡易実装 |
