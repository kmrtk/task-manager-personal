# CLAUDE.md — Claude Code 運用ルール

## GitHub フロー（必ず守ること）

### 作業の流れ

1. **Issue を作成する** — 作業を始める前に必ず GitHub Issue を作る
2. **ブランチを切る** — Issue 番号を含むブランチ名で作業ブランチを作成する
3. **実装する** — ブランチ上でコミットを積む
4. **PR を作成する** — master へのマージは必ず Pull Request 経由で行う
5. **master への直接プッシュは禁止**

### ブランチ命名規則

```
feature/#{Issue番号}-短い説明   # 新機能
fix/#{Issue番号}-短い説明       # バグ修正
docs/#{Issue番号}-短い説明      # ドキュメント
```

例：
- `feature/#12-task-read-api`
- `fix/#7-folder-null-error`
- `docs/#3-api-spec`

### Issue の書き方

- **タイトル**: 何をするか一言で（例: `タスク読み取りAPIを実装する`）
- **本文**: なぜ必要か、何をするか、完了条件を書く

### コミットメッセージ

```
#{Issue番号} 変更内容の要約
```

例: `#12 Task エンティティとリポジトリを追加`

### PR の書き方

- タイトルに Issue 番号を含める（例: `#12 タスク読み取りAPI実装`）
- 本文に `Closes #12` を書くと Issue が自動クローズされる

## ポート管理ルール（必ず守ること）

このプロジェクトで使用するポートは以下の通り固定する。別ポートでの起動は禁止。

| サービス | ポート |
|---|---|
| nginx（フロントエンド） | 80 |
| Spring Boot（バックエンド） | 8080 |
| PostgreSQL | 5432 |

### ポート競合が発生した場合の対処

別のプロセスがポートを使用中でサービスが起動できない場合は、**そのプロセスを停止してから**指定ポートで起動する。別ポートへの変更は行わない。

```powershell
# 使用中のポートを確認（例：8080）
netstat -ano | findstr :8080

# PIDを確認してプロセスを停止
taskkill /PID {PID番号} /F
```

ポートを変更すると nginx のプロキシ設定や application.yml との整合性が崩れるため、
設定ファイルを書き換えてのポート変更は行わないこと。

## Claude Code への指示

- 実装作業を始める前に、対応する Issue が存在するか確認すること
- Issue がなければ作成を提案すること
- ブランチは必ず上記の命名規則に従って作成すること
- master へ直接コミット・プッシュしないこと
- ポート競合が発生した場合は競合プロセスを停止し、必ず指定ポートで起動すること
